# 2026-02-06 菜单页购物车状态与删除接口异常复盘

## 1. 事件概述
- 发生页面：`/menu`（点餐页）
- 主要现象：
  - 未登录时页面仍显示购物车角标和弹窗历史商品；
  - 点击加购会被要求登录（跳转登录页）；
  - 商品数量为 `1` 时点击减号，出现“操作失败，请重试”，Network 显示 `500`。

---

## 2. 问题一：未登录却显示历史购物车

### 2.1 现象
- 刚启动前端后进入点餐页，悬浮购物车显示历史数量和商品；
- 但执行加购/减购时又被判定未登录并跳转登录页；
- 呈现为“登录状态”和“购物车展示状态”不一致。

### 2.2 根因
- 购物车本地缓存使用全局键 `dawn_dusk_cart_items`，且初始化时无条件读取；
- `auth` 登出/鉴权失效时只清理 token/user，没有同步清理购物车缓存；
- 菜单页仅在已登录时拉取服务端购物车，未登录时不会覆盖本地残留。

### 2.3 前端修复
- 购物车缓存改为按用户隔离：`dawn_dusk_cart_items_v2:{userId}`；
- 新增 `cartStore.bindUser(userId | null)`：
  - 登录绑定用户，读取对应用户缓存；
  - 未登录绑定 `null`，清空内存购物车；
- 新增 `cartStore.clearUserCache()`，登出/失效时同步清理；
- `auth` 登录/登出流程接入购物车绑定与清理；
- 应用启动时先做 `auth/cart` 对齐，再挂载应用；
- 菜单页悬浮购物车调整为仅登录可见：`v-if="authStore.isLoggedIn"`；
- 清理旧键 `dawn_dusk_cart_items` 避免历史脏数据继续生效。

### 2.4 结果
- 未登录进入 `/menu` 不再显示历史购物车；
- 登录后以服务端购物车为准；
- 多账号同设备不再串购物车数据。

---

## 3. 问题二：数量 1 再减触发 500

### 3.1 现象
- `2 -> 1` 正常；
- `1 -> 0`（触发删除单品）报错；
- 控制台报：`Failed to load resource: 500 (Internal Server Error)`。

### 3.2 前端抓包证据（DevTools）
- 失败接口：`DELETE /api/member/cart/items/{productId}`
- 返回：`{"code":5000,"message":"系统内部错误","data":null}`
- `POST /api/member/cart/items`、`PUT /api/member/cart/items/{id}` 正常。

### 3.3 后端确认根因
- 删除链路此前使用逻辑删除；
- 命中 `t_cart_item` 唯一索引冲突（历史软删数据导致）；
- 因此返回 `500`。

### 3.4 后端修复内容（后端同学反馈）
- 删除链路改为物理删除（`removeItem/removeItems/clearCart` 统一）；
- `t_cart_item` 迁移为无软删设计（迁移脚本：`V8__cart_item_remove_soft_delete.sql`）；
- 删除不存在商品时改为业务错误 `400100`（HTTP 400），不再返回 500；
- 后端定向测试通过：`CartControllerTest`、`CartServiceImplTest`（10/10）。

### 3.5 前端兼容性加固
- 购物车 store 增加兜底同步：
  - 当 `add/update/remove` 返回体不是完整 `CartResponse` 时，自动执行 `getCart()` 强制同步；
  - 避免接口返回结构不一致导致角标/弹窗不刷新。

### 3.6 联调复测结果
- 在同一环境复测 `2 -> 1 -> 0`：
  - `PUT /api/member/cart/items/{id}`：`200`
  - `DELETE /api/member/cart/items/{id}`：`200`
  - 后续 `GET /api/member/cart`：`200`
- 未再出现 500；
- 悬浮角标与弹窗内容同步正确。

---

## 4. 最终结论
- 本次是“前后端各有一处问题叠加”：
  - 前端：登录态与购物车缓存未绑定，导致展示错态；
  - 后端：删除单品接口因软删历史数据触发索引冲突导致 500。
- 目前已完成修复并联调通过，问题关闭。
