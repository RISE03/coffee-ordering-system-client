# 咖啡厅在线点餐与消费积分系统
技术栈与架构选型说明（V1.4）
## 1 总体架构概述
1 总体架构概述

本系统采用典型的前后端分离架构：

- 前端：基于 Vue 3 + Vite 构建的单页应用（SPA），负责页面展示与交互逻辑；
- 后端：基于 Spring Boot 3.5.8 的 RESTful 服务，负责业务逻辑处理和数据持久化；
- 数据库与中间件：使用 MySQL 8.0.43 作为关系型数据库，Redis 作为缓存与会话辅助组件；
- 接口规范：通过 Springdoc OpenAPI + Knife4j 生成接口文档，前后端通过统一的 REST API 进行交互。

整体架构目标：在保证功能完整与性能可靠的前提下，使用当前主流、易于维护且适合毕业设计展示的技术栈。
## 2 后端技术栈选型
2 后端技术栈选型

2.1 开发语言与运行环境
- 开发语言：Java
- 推荐 JDK 版本：JDK 21（LTS），保证与 Spring Boot 3.5.8 的兼容性与长期支持。
注：本机可安装更高版本 JDK，但本项目建议选择 JDK 21 作为项目 SDK。

2.2 主框架：Spring Boot 3.5.8

- 用途：快速构建独立运行的后端应用，简化依赖管理与配置。
- 使用模块：
- spring-boot-starter-web：提供 RESTful Web 服务能力；
- spring-boot-starter-validation：提供参数校验（JSR 303/380）；
- spring-boot-starter-security：提供认证与鉴权基础能力；
- spring-boot-starter-data-redis：提供访问 Redis 的能力；
- spring-boot-starter-test：用于单元测试与集成测试。

2.3 安全与认证：Spring Security + JWT
- Spring Security：用于实现统一的认证鉴权过滤链，对接口访问进行权限控制；
- JWT（JSON Web Token）：作为前后端分离项目的登录令牌形式，登录成功后由后端生成 token，前端在请求头中携带。

主要设计要点：
- 登录接口返回 JWT，包含用户 ID、角色等信息；
- 前端在请求头中添加 Authorization: Bearer <token>；
- 后端通过过滤器解析 JWT，将用户信息注入安全上下文；
- 按角色对接口进行访问控制，例如：
- /api/admin/** 仅管理员可访问；
- /api/staff/** 仅店员可访问；
- /api/** 默认登录后可访问，部分如 /api/auth/** 允许匿名访问。

2.4 数据访问层：MyBatis-Plus
- 用途：在 MyBatis 的基础上提供大量 CRUD 封装，简化单表操作；
- 优点：
- 提供通用 Mapper 和 Service，减少重复代码；
- 保留编写 SQL 的灵活性，适用于实际项目；
- 社区资源丰富，文档完善，便于学习与排错。

典型用法：
- 实体类与数据表一一对应，命名规范，如：User -> t_user；
- Mapper 接口继承 BaseMapper<T>，无需手写基础 CRUD；
- 利用分页插件简化分页查询。

2.5 数据库访问驱动
- MySQL Connector/J：作为 Java 应用访问 MySQL 数据库的官方驱动。

2.6 数据库版本管理：Flyway
- 用途：管理数据库结构的版本演进（如建表、加字段、加索引等）；
- 使用方式：
- 在 resources/db/migration 目录下维护 SQL 脚本；
- 初始脚本命名为 V1__init_schema.sql，用于创建全部业务表；
- 后续变更新增 V2__xxx.sql、V3__xxx.sql 等版本脚本；
- 应用启动时自动执行尚未执行的脚本，保证不同环境数据库结构一致。

2.7 缓存组件：Redis
- 用途：
- 存储暂时性数据（如验证码、临时 token 等）；
- 缓存高频访问的静态数据（如商品分类列表、热门商品等）；
- 为后续扩展分布式会话与限流等功能预留空间。
- 接入方式：通过 Spring Data Redis 统一访问。

2.8 接口文档：Springdoc OpenAPI + Knife4j
- Springdoc OpenAPI：自动扫描 Controller 与注解，生成 OpenAPI 规范文档；
- Knife4j：基于 OpenAPI 的增强 UI 界面，提供更友好的接口调试与文档浏览体验；
- 作用：
- 方便前后端联调，前端可直接在文档界面查看参数与示例；
- 在毕业答辩时可作为系统接口说明的可视化展示。
## 3 前端技术栈选型
3 前端技术栈选型

3.1 主框架：Vue 3
- 用途：构建前台会员端、店员端与管理后台页面；
- 选型理由：
- 学习曲线相对平滑，中文社区与资料丰富；
- 与 Element Plus、Pinia、Vue Router 等生态配合紧密；
- 适合构建中小型后台管理与业务系统。

推荐使用 Composition API，便于逻辑复用和代码组织。

3.2 构建工具：Vite
- 用途：前端工程的构建与开发服务器；
- 优点：
- 冷启动速度快，开发体验好；
- 配置相对简单，适合毕业设计项目。

3.3 路由管理：Vue Router 4
- 用途：管理 SPA 的前端路由与页面跳转；
- 典型设计：
- /login：登录页面；
- /member/**：会员端页面（如首页、商品列表、订单、积分中心等）；
- /staff/**：店员端页面（如待接单、制作中、待取餐等）；
- /admin/**：后台管理页面（如商品管理、订单管理、用户管理等）。

3.4 状态管理：Pinia
- 用途：管理全局状态，如：
- 当前登录用户信息（id、昵称、角色、等级等）；
- JWT token；
- 当前 UI 主题（朝/暮模式）；
- 选型理由：API 简洁、类型支持友好，适合 Vue 3 项目。

3.5 UI 组件库：Element Plus（frontend-admin） / Naive UI（frontend-client）
- 用途：快速构建表单、表格、弹窗、菜单、分页等常见界面组件；
- 场景：
- 管理后台：大量使用 Element Plus 组件构建数据管理界面；
- 店员端：使用表格、卡片等组件展示订单列表；
- 前台会员端：可结合自定义样式与 Naive UI 组件实现统一风格。

3.6 网络请求：Axios
- 用途：前端统一发起 HTTP 请求，调用后端 REST API；
- 设计要点：
- 封装 request 实例，统一配置 baseURL（如 /api）；
- 在请求拦截器中自动添加 token 到请求头；
- 在响应拦截器中统一处理错误码与提示信息。

3.7 样式与主题：Tailwind CSS + 自定义样式
- Tailwind CSS：作为实用类优先的样式框架，帮助快速构建布局和响应式页面；
- 自定义样式：
- 使用 CSS 变量（如 --primary-color、--bg-color、--text-color）实现“朝/暮”主题动态切换；
- 配合少量 SCSS 或自定义 CSS，覆盖 Element Plus 默认主题以匹配品牌配色方案。

3.8 图标与可视化（可选）
- 图标库：可选用 Element Plus 自带图标或第三方图标库；
- 图表展示（如订单统计、销量统计）：可选用 ECharts 或其他轻量级图表库。
## 4 数据库与中间件
4 数据库与中间件

4.1 关系型数据库：MySQL 8.x
- 用途：存储核心业务数据（用户、商品、订单、积分、优惠券等）；
- 选型理由：
- 开源、成熟，兼容性好；
- 社区与文档丰富，适合作为毕业设计数据库。

4.2 缓存：Redis
- 用途：
- 缓存热点数据（如商品分类、首页推荐列表等）；
- 存储临时数据（如验证码、限流计数器等）；
- 部署方式：单实例部署即可满足毕设项目需求。

4.3 数据库版本管理：Flyway
- 见“2.6 数据库版本管理：Flyway”。

## 5 开发工具与环境
5 开发工具与环境

5.1 开发工具
- 后端 IDE：IntelliJ IDEA 或 Eclipse（推荐 IntelliJ IDEA）；
- 前端编辑器：VS Code 或 WebStorm 等；
- 构建工具：
- 后端：Maven 或 Gradle（推荐 Maven）；
- 前端：Vite（内部使用 Rollup/esbuild 等工具链）；
- 版本控制：Git（可使用 GitHub / Gitee 仓库存放代码）。

5.2 运行环境（建议）
- 后端：
- JDK：21
- Spring Boot：3.5.8
- 操作系统：Windows / Linux / macOS 均可
- 前端：
- Node.js：推荐使用 Node 22 LTS（例如 22.21.1），也兼容 20 LTS / 24 Current。Windows + npm 环境下，使用 Vite 初始化项目前端项目时，可能因 npm 对 optionalDependencies 的已知问题导致 Rollup 平台依赖未正确安装，出现类似 “Cannot find module '@rollup/rollup-win32-x64-msvc'” 的错误；遇到该问题时，可先删除 node_modules 目录和 package-lock.json 文件后重新执行 npm install，如仍存在错误，再手动安装缺失依赖：npm install @rollup/rollup-win32-x64-msvc --save-dev。同时，Vite 开发服务器建议在配置文件中通过 server.host = '0.0.0.0' 进行统一配置，避免每次通过命令行追加 --host 参数：frontend-admin（管理端）默认使用 5173 端口，frontend-client（用户端）默认使用 5174 端口，以便在同一开发机上同时调试两个前端项目，并支持通过本机 IP 在局域网内访问。
- npm 或 pnpm 作为包管理工具

5.3 部署方式（示例）
- 开发环境：
- 后端直接在 IDE 中以 Spring Boot 应用方式运行；
- 前端通过 Vite 启动开发服务器（npm run dev）；
- 生产/演示环境：
- 后端打包为可执行 jar，在服务器上运行；
- 前端项目打包为静态文件（npm run build），通过 Nginx 或 Spring Boot 静态资源路径部署；
- 前后端可通过反向代理或同源方式进行联通。
## 6 技术选型特点总结
6 技术选型特点总结

- 主流性：
- 后端基于 Spring Boot 3.5.8 + Spring Security + MyBatis-Plus，是当前 Java 生态开发业务系统的主流组合；
- 前端基于 Vue 3 + Vite + Element Plus，是常见的中后台与业务系统前端技术栈。

- 可维护性：
- 前后端分离，职责清晰，便于后续扩展或迁移；
- 使用 Flyway 管理数据库结构变更，降低手工改表风险；
- 使用 OpenAPI/Knife4j 生成接口文档，方便前后端协作。

- 展示性：
- 动态主题（朝/暮模式）结合 Tailwind CSS 和 Element Plus，可以在答辩中直观展示系统的视觉设计能力；
- 时间段推荐、积分（光阴值）、会员等级等业务逻辑能够体现系统的业务深度和创新点。

本技术栈在保证学习成本可控的前提下，兼顾了工程实践价值与毕业设计展示效果，适合用于本系统的实现与后续论文撰写。

### 5.2.1 Vite 开发服务器配置示例（frontend-admin / frontend-client）

为方便在开发环境中直接通过 `npm run dev` 启动并在局域网访问，本项目对两个前端子项目的 Vite 开发服务器进行统一配置。

#### （1）frontend-admin（管理端）vite.config.js 示例

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// 管理端 frontend-admin 的 Vite 配置
export default defineConfig({
  plugins: [vue()],
  server: {
    host: '0.0.0.0', // 支持通过本机 IP 在局域网访问，无需每次追加 --host
    port: 5173,      // 管理端默认端口
    open: true,      // 启动时自动在浏览器打开（如不需要可改为 false）
  },
  // 后续如需本地开发代理后端接口，可在此处按需开启
  // server: {
  //   proxy: {
  //     '/api': {
  //       target: 'http://localhost:8080',
  //       changeOrigin: true,
  //     },
  //   },
  // },
})
```

#### （2）frontend-client（用户端）vite.config.js 示例

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// 用户端 frontend-client 的 Vite 配置
export default defineConfig({
  plugins: [vue()],
  server: {
    host: '0.0.0.0', // 同样支持局域网访问
    port: 5174,      // 用户端默认端口，与管理端区分开
    open: true,
  },
  // 如需代理到后端接口，可在此处按需开启
  // server: {
  //   proxy: {
  //     '/api': {
  //       target: 'http://localhost:8080',
  //       changeOrigin: true,
  //     },
  //   },
  // },
})
```
