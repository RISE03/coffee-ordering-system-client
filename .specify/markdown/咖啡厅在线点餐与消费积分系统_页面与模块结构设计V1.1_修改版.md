# 咖啡厅在线点餐与消费积分系统
页面与模块结构设计（V1.1）

> 本版本在 V1.0 基础上：
> - 补充了主页“时间段推荐区”和“朝/暮主题”相关说明；
> - 将订单相关路由参数统一为 `orderId`，以便与接口设计保持一致；
> - 补充后台“商品时间段配置页面（可选扩展）”描述；
> - 对积分（光阴值）、优惠券、订单详情与评价等部分页面元素进行了细化说明。

## 前台（游客/会员）页面结构
1 前台（游客/会员）页面结构

1.1 登录 / 注册模块
1.1.1 登录页面（/login）
- 功能：为已注册用户提供身份认证入口。
- 主要元素：
- 输入：账号（用户名，可使用手机号注册后作为用户名）、密码。
- 按钮：登录、跳转注册。
- 反馈：错误信息提示（账号不存在、密码错误等）。

1.1.2 注册页面（/register）
- 功能：为新用户提供注册入口。
- 主要元素：
- 输入：账号（用户名，支持使用手机号作为用户名）、密码、确认密码；可选单独填写手机号字段。
- 校验：账号唯一性、两次密码一致性、基本格式校验。
- 按钮：提交注册、返回登录。

1.2 首页与商品浏览模块
1.2.1 首页 / 商品列表页（/ 或 /products）
- 功能：展示咖啡厅菜单入口，是用户的主要浏览和点餐入口。
- 主要区域：
- 顶部导航：LOGO、首页、我的订单、积分中心、个人中心、登录/退出等入口。
- 主题切换（朝/暮模式）：右上角提供主题切换控件（如 ☀️ / 🌙 图标），默认根据时间自动应用“朝/暮”主题，用户也可手动切换并在本地记忆。
- 时间段推荐区（时间感菜单）：根据当前时间段展示“朝·醒 / 午·憩 / 暮·微醺”等推荐商品区块，以卡片形式展示推荐商品，点击可加入购物车或查看详情。
- 左侧分类栏：咖啡、饮品、甜点、小食等分类列表。
- 右侧商品列表：以卡片形式展示商品图片、名称、价格、简介，“加入购物车”按钮。
- 搜索框：支持按关键字搜索商品名称。

1.2.2 商品详情页（/product/:id）
- 功能：展示单个商品的详细信息，为用户提供下单决策依据。
- 主要元素：
- 商品图片、大标题、价格。
- 商品详细介绍（口味说明、原料等）。
- 可选规格（如大小杯、冷热等，根据实际实现决定；当前版本可先按单一规格实现，后续可扩展为规格管理）。
- 加入购物车区域：数量选择、“加入购物车”按钮。
- 用户评价列表（可选）：展示评分与部分评价内容；如当前版本后端未实现评价查询接口，则可暂不展示，仅在订单详情中提供“去评价”入口。

1.3 购物车与结算模块
1.3.1 购物车页面（/cart）
- 功能：展示用户已选购的商品，支持修改、删除和跳转结算。
- 主要元素：
- 商品表格：商品名称（可跳转详情）、单价、数量（可调整）、小计、删除按钮。
- 底部汇总：合计金额、“去结算”按钮。
- 状态提示：当购物车为空时显示引导文案（如“购物车空空如也，去逛逛吧~”）。

1.3.2 结算 / 下单页面（/checkout）
- 功能：对购物车中的商品进行最终确认，并收集取餐/收货信息，生成订单。
- 主要区域：
- 商品确认区：只读展示本次订单的商品列表及金额小计。
- 取餐方式区：单选“到店自取 / 外卖配送”；
- 到店自取：输入/选择取餐人姓名、手机号。
- 外卖配送：选择已有收货地址或新增地址。
- 优惠信息区：
- 可用优惠券下拉列表（仅显示当前订单金额满足使用门槛的优惠券）。
- 显示所选优惠券的优惠额度与条件。
- 金额汇总区：原价总额、优惠金额、实付金额。
- 操作区：“提交订单”按钮，提交成功后跳转订单详情页（状态为“待支付”）。

1.4 订单模块
1.4.1 订单列表页（/orders）
- 功能：展示当前登录用户的全部订单及按状态筛选。
- 主要元素：
- 状态筛选 Tab：全部、待支付、已支付/待接单、制作中、待取餐、已完成、已取消、退款处理中、已退款等。
- 订单列表项：订单号、下单时间、实付金额、当前状态。
- 操作按钮：
- 待支付：去支付、取消订单。
- 已支付/待接单：申请退款（如实现）。
- 其他状态：查看详情。

1.4.2 订单详情页（/orders/:orderId）
> 说明：这里路由参数使用 `orderId`，与接口 `/api/orders/{orderId}` 对齐，页面中同时展示业务单号 `orderNo`。
- 功能：展示单个订单的详细信息，支持相应状态下的操作。
- 主要元素：
- 基本信息：订单号、下单时间、当前状态、支付时间（如有）。
- 取餐信息：取餐方式、取餐人信息或收货地址等。
- 商品明细：商品名称、数量、单价、小计。
- 金额信息：原价总额、优惠金额、实付金额。
- 优惠券信息：若使用优惠券则展示名称、满减规则等。
- 积分信息：若订单为“已完成”，展示本单获得的积分数量。
- 操作区：
- 待支付：去支付、取消订单。
- 已支付/待接单：申请退款。
- 已完成但未评价：去评价。

1.5 积分与优惠券模块
1.5.1 积分中心页面（/points）
- 功能：展示会员积分信息、积分记录以及积分兑换入口。
- 主要元素：
- 顶部概览：当前积分余额、累计获得积分。
- Tab 切换：
- 积分记录：列表展示时间、类型（消费获得/兑换优惠券/退款扣减等）、变动积分值、关联订单/优惠券，按时间倒序排列。
- 积分商城：展示可用积分兑换的优惠券模板列表，包括名称、满减条件、所需积分、有效期说明等。
- 操作按钮：“立即兑换”，积分不足时按钮禁用或弹出提示。

1.5.2 我的优惠券页面（/coupons）
- 功能：展示会员持有的优惠券列表。
- 主要元素：
- 状态 Tab：未使用、已使用、已过期。
- 优惠券卡片：名称、满减条件、面值或折扣、有效期、当前状态。
- 提示：提示用户在下单时可选择使用未使用且未过期的优惠券。

1.6 地址与个人信息模块
1.6.1 收货地址管理页面（/addresses）
- 功能：管理用户的多个收货地址，用于外卖配送场景。
- 主要元素：
- 地址列表：收货人姓名、手机号、详细地址、是否默认。
- 操作按钮：新增地址、编辑、删除、设为默认。
- 地址编辑弹窗/页面：表单校验（必填项、手机号格式等）。

1.6.2 个人中心页面（/profile）
- 功能：提供账号相关的信息展示和入口。
- 主要元素：
- 基本信息：头像、昵称、手机号（支持修改）。
- 快捷入口：我的订单、积分中心、我的优惠券、收货地址管理。
- 安全设置（可选）：修改密码入口。

1.7 评价模块
1.7.1 订单评价页面或弹窗（/orders/:orderId/review 或详情页弹窗）
> 说明：评价路由同样使用 `orderId` 作为路径参数。
- 功能：对已完成订单进行评价。
- 主要元素：
- 星级评分控件（1~5 星）。
- 文本评价输入框。
- 提交按钮：提交后不可重复评价，可在订单详情页展示评价结果。

## 店员端页面结构
2 店员端页面结构

2.1 店员登录页面（/staff/login）
- 功能：为店员提供登录入口，区分于普通用户和管理员登录。
- 主要元素：
- 输入：账号、密码。
- 按钮：登录。
- 反馈：登录失败时的错误提示。

2.2 今日订单列表页面（/staff/orders/today）
- 功能：展示店员当前值班需要处理的订单列表，支持按状态查看。
- 主要元素：
- 状态筛选：待接单、制作中、待取餐、已完成。
- 订单列表项：订单号、下单时间、取餐方式（自取/外卖）、顾客姓名或取餐人信息、订单金额、当前状态。
- 操作按钮：
- 对“已支付/待接单”订单：显示“接单”按钮。
- 对“制作中”订单：显示“制作完成”按钮。
- 对“待取餐”订单：显示“确认取餐”按钮。

2.3 店员订单详情页面（/staff/orders/:orderId）
- 说明：路径参数为 orderId，与前台和接口保持一致。
- 功能：帮助店员查看单个订单的详细信息并进行处理。
- 主要元素：
- 基本信息：订单号、状态、下单时间、取餐方式、取餐人信息或收货地址。
- 商品明细：商品名称、数量、备注等。
- 操作区：根据订单状态展示对应操作按钮（接单、制作完成、确认取餐）。

## 管理后台页面结构
3 管理后台页面结构

3.1 管理员登录页面（/admin/login）
- 功能：提供管理员登录入口。
- 主要元素：
- 输入：管理员账号、密码。
- 按钮：登录。
- 反馈：错误提示。

3.2 仪表盘页面（/admin/dashboard）
- 功能：展示系统的整体运行情况概要。
- 主要元素：
- 数据卡片：今日订单数、今日销售额、本周/本月销售概览等。
- 简要图表（可选）：按日统计的订单数量/销售额折线图，商品销量排行柱状图。

3.3 商品分类管理页面（/admin/categories）
- 功能：管理商品分类信息。
- 主要元素：
- 列表：分类名称、排序、状态（启用/停用）。
- 操作：新增分类、编辑分类、启用/停用、删除分类。
- 校验：删除前如存在关联商品，则禁止删除并提醒。

3.4 商品管理页面（/admin/products）
- 功能：管理商品基础信息。
- 主要元素：
- 查询条件：分类、商品名称关键字、上架状态。
- 列表字段：图片缩略图、商品名称、分类、价格、库存、上架状态。
- 操作：新增商品、编辑商品、上架/下架、调整库存。
- 新增/编辑页面/弹窗：填写名称、价格、简介、所属分类、库存、上传图片、上下架状态等。
- 调整库存弹窗：输入调整数量（正负）、调整原因，记录到操作日志。

3.4.1 商品时间段配置页面（可选扩展）（/admin/product-time-slots 或商品编辑页子模块）
- 功能：配置不同时段显示/推荐的商品，用于实现“朝·醒 / 午·憩 / 暮·微醺”等时间段菜单和推荐。
- 主要元素：
- 查询条件：商品名称、分类、时间段。
- 列表字段：商品名称、所属分类、适用时间段（早/中/晚等）、推荐级别（可选）、启用状态。
- 操作：为商品新增/编辑/删除时间段配置。实现形式可以是独立页面，也可以集成在商品编辑弹窗中的“时间段推荐”区域。

3.5 订单管理页面（/admin/orders）
- 功能：统一查看和管理所有订单。
- 主要元素：
- 查询条件：订单号、用户账号、状态、下单时间范围。
- 列表字段：订单号、用户、下单时间、实付金额、当前状态。
- 操作：查看订单详情、处理退款申请。

3.5.1 后台订单详情页面（/admin/orders/:orderId）
- 说明：路径参数为 orderId，与接口和前台路由保持一致，页面中展示业务单号 orderNo。
- 功能：展示订单全量信息，并提供退款审核等操作入口。
- 主要元素：
- 基本信息：订单号、用户、状态、下单时间、支付时间、取餐方式等。
- 商品明细：商品名称、数量、单价、小计等。
- 金额信息：原价总额、优惠金额、实付金额。
- 优惠与积分：是否使用优惠券、对应模板；本单发放或扣减的积分信息。
- 状态变更记录：状态从何时由谁修改为何状态。
- 退款处理区：当订单处于“退款处理中”状态时，展示“同意退款”和“拒绝退款”按钮，以及填写备注的输入框。

3.6 会员管理页面（/admin/members）
- 功能：管理会员用户的账号及查看相关数据。
- 主要元素：
- 查询条件：账号、手机号等。
- 列表字段：账号、昵称、手机号、状态（启用/禁用）、累计消费金额、当前积分。
- 操作：查看详情、启用/禁用账号。

3.6.1 会员详情页面（/admin/members/:userId）
- 内容：展示该会员的详细信息，包括基本信息、订单列表和积分记录等。

3.7 店员账号管理页面（/admin/staff）
- 功能：维护店员账号。
- 主要元素：
- 列表字段：账号、姓名（可选）、角色、状态。
- 操作：新增店员、重置密码、启用/禁用。

3.8 积分规则配置页面（/admin/points-rule）
- 功能：配置积分获取规则。
- 主要元素：
- 输入项：每 1 元实付对应的积分数量（如 1 元=1 积分）。
- 说明：规则变更仅对变更后新产生并完成的订单生效，不追溯历史记录。
- 保存按钮：提交并持久化规则。

3.9 优惠券模板管理页面（/admin/coupon-templates）
- 功能：配置可供用户兑换的优惠券模板。
- 主要元素：
- 列表字段：名称、门槛金额、减免金额、所需积分、有效期（领取后 N 天）、状态（启用/停用）。
- 操作：新增模板、编辑模板、启用/停用。
- 新增/编辑弹窗：填写模板相关字段并校验输入合法性。

3.10 系统配置页面（/admin/settings）
- 功能：管理系统级基础信息。
- 主要元素：
- 店铺信息：门店名称、联系电话、门店地址。
- 营业时间说明：例如 9:00-22:00。
- 系统公告：展示在前台首页或个人中心的公告文本。

3.11 日志查看页面（可选）（/admin/logs）
- 功能：供管理员查看关键操作日志记录。
- 主要元素：
- 查询条件：操作人、操作类型（库存调整、积分调整、订单状态修改等）、时间范围。
- 列表字段：操作时间、操作人、操作类型、目标对象标识（如订单号、用户 ID）、描述信息。
