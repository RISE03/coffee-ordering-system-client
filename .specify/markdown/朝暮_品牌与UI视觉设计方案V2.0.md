# 基于 SpringBoot + Vue 的「朝暮」空间在线点餐与积分系统  
品牌与 UI 视觉设计方案（V2.0）

> **版本说明**：  
> - V1.1：完成品牌核心设定、基础配色与时间感菜单、积分体系等设计。  
> - **V2.0：在 V1.1 基础上，补充并细化了客户端首页在「日间（Dawn）」与「夜间（Dusk）」双模式下的 UI 布局与玻璃拟态视觉方案。**

## 1 品牌核心设定（Brand Identity）

### 1.1 品牌名称

- 中文品牌：**朝暮**  
- 英文品牌：**Dawn & Dusk**

设计寓意：取意“朝朝暮暮”。Dawn & Dusk 的缩写 **D&D** 具有对称美，同时也代表从日出到日落的全天候陪伴，与咖啡厅“从清晨到夜晚”的开放时段、陪伴场景天然契合。

### 1.2 Logo 概念

这个方案更强调氛围感和与您现代毛玻璃 UI 风格的融合，用光影的变化来体现时间。

流光时刻 (The Flowing Light)

核心构图：一个柔和、流动的圆形光晕作为背景。

色彩寓意：光晕的色彩采用了大胆的对撞渐变。左上方是温暖、明亮的暖金色光辉（象征晨光/朝），柔和地过渡到右下方的深邃幽蓝色调（象征夜幕/暮）。

核心形态：在流动的光影之中，用一条抽象、连贯的白色线条，勾勒出一个富有动感的咖啡杯和袅袅升起的热气的形态。这个形态仿佛是由光线汇聚而成，轻盈而富有灵性。

风格：现代、梦幻、富有艺术感。它就像 UI 背景上的一个发光体，完美契合“光阴”的主题，给人一种温暖而沉浸的感觉。

### 1.3 品牌口号（Slogan）

- **主标语**：朝暮与共，不止咖啡。  
- **副标语**：把时间，浪费在美好的朝暮里。

说明：在系统 UI、登录页、积分中心、订单完成页等位置，可以灵活使用主 / 副标语作为文案，强化品牌记忆点。

## 2 UI 视觉与配色方案（Visual Design）

系统采用 **Vue 前端 + CSS Variables** 实现动态主题（Dynamic Theme），根据系统时间自动切换“朝 / 暮”两种模式；同时在界面右上角提供手动切换按钮（☀️ / 🌙），便于演示和满足用户自由选择偏好的需求。

在 V2.0 中，首页进一步采用 **毛玻璃拟态（Glassmorphism）** 视觉语言，通过半透明磨砂卡片 + 实景背景的方式，强化“在真实咖啡馆空间里点单”的沉浸感。

### 2.1 整体设计思路

- **关键词**：时间感、温度、陪伴、沉浸。  
- **视觉目标**：  
  - 在清晨与白天营造“温暖、提神”的感觉；  
  - 在傍晚与夜间传达“安静、微醺、陪伴”的氛围。  
- **材质与空间**：  
  - 首页与主要前台页面采用玻璃拟态卡片悬浮于**实景咖啡厅背景**之上；  
  - 利用模糊、投影与柔和描边制造层次感与空气感。  
- **技术实现**：  
  - 通过 root 级 CSS 变量（如 `--primary-color`、`--bg-color`、`--text-color`）统一控制颜色；  
  - 使用 `transition` 为颜色、背景模糊等属性增加过渡动画，实现 Dawn/Dusk 模式平滑切换。

### 2.2 配色方案：流金岁月（主推，侧重“太阳与温度”）

核心逻辑：**朝是金色的暖阳，暮是深邃的夜空。**  
通过强烈的冷暖对比，使咖啡、饮品、甜点在视觉上更具食欲感和记忆点。

#### 2.2.1 朝模式（Dawn Mode）[06:00 - 17:59]

- **主色（Primary）**：琥珀黄 `#F5B041`  
  象征初升的太阳，温暖、醒目，适合作为按钮、标签、价格高亮颜色。
- **背景（Background）**：舒芙蕾白 `#FCF9F2`  
  极淡的暖米色，类似牛奶的柔和质感，适合作为整体页面背景色。
- **文字（Text）**：焦糖褐 `#5D4037`  
  代替生硬的纯黑，更有拿铁与咖啡的质感，阅读舒适。

**配合建议：**

- 商品卡片：采用白色 / 浅暖色卡片背景，搭配细边框和阴影；鼠标悬停时，主色加深并略微放大。  
- 价格与折扣：使用琥珀黄作为醒目强调色，突出促销与优惠信息。

#### 2.2.2 暮模式（Dusk Mode）[18:00 - 05:59]

- **主色（Primary）**：月光银 `#D5D8DC` 或淡金 `#FAD7A0`  
  用作按钮描边、高亮边框或图标点缀，模拟夜色中的灯光与余晖。
- **背景（Background）**：黛蓝 `#1A2530`  
  接近黑色的深蓝，静谧克制，适合作为夜间模式的主背景色。
- **文字（Text）**：云峰白 `#ECF0F1`  
  保证夜间阅读清晰度，同时避免纯白的刺眼，可配合次级文字使用略深的灰蓝色。

**配合建议：**

- 页面主内容区可使用稍浅的深蓝卡片（如 `#1E2933`），在深背景上形成层次感。  
- 卡片边角适度圆角 + 阴影，以保证后台表格 + 前台卡片都具有统一的视觉风格。  
- 保留足够的对比度，尤其在手机端注意字体大小与颜色组合，确保可读性。

### 2.3 前台与后台的风格统一与差异

- **前台（用户点餐端）**：  
  - 更强调情绪与氛围，可使用大面积背景图、渐变、插画、品牌标语等；  
  - Dawn 模式适合展示早餐与咖啡，Dusk 模式适合展示特调饮品与夜间饮品。  

- **店员端与管理后台**：  
  - 以简洁、易读为主，主体区域采用较浅的灰白色背景，搭配 Element Plus 默认样式；  
  - 主色仅作为导航栏、高亮按钮、选中菜单的点缀色，减少干扰，使表格与数据清晰呈现。

### 2.4 客户端首页 UI 布局设计（Dawn & Dusk 双模式）

本小节对「朝暮」客户端首页在 **日间模式（Dawn）** 与 **夜间模式（Dusk）** 下的视觉风格与功能布局进行详细定义。两种模式在**信息结构**与**交互逻辑**上保持一致，仅通过配色、材质与内容侧重点实现主题切换。

#### 2.4.1 页面结构总览

首页整体采用「实景背景 + 毛玻璃卡片」结构，自上而下划分为五个区域：

1. 顶部状态与导航栏（Top Bar）  
2. 头部推荐区（Hero Banner）  
3. 分类导航栏（Category Tabs）  
4. 商品列表区（Product Grid）  
5. 底部导航栏（Bottom Navigation）

所有前景元素以不同透明度的玻璃卡片形式悬浮于背景之上，保留一定留白，保证呼吸感与可读性。

#### 2.4.2 日间模式首页 UI 设计（Dawn Mode）

##### （1）设计概述

- **模式名称**：日间模式（Dawn Mode）  
- **主题**：朝·醒神（Morning Awake）  
- **设计目标**：传达清晨与白昼的温暖、明亮与活力，配合品牌“把时间，浪费在美好的朝暮里”的理念，为用户营造清爽、自然的点餐体验。

背景采用阳光充足的咖啡厅室内场景，透过落地窗可看到明亮城市天际线，营造早晨充满活力的氛围。

##### （2）视觉风格与氛围

- 氛围关键词：**温暖、通透、自然光感、沉浸式**。  
- UI 材质：大面积高透明度白色毛玻璃卡片，带轻微暖色光泽与柔和背景模糊，使界面轻盈且富有层次。

##### （3）配色方案（Dawn 首页）

在 2.2.1 基础上，对首页主要元素配色做进一步约定：

- **容器背景**：高透明度白色毛玻璃（卡片）+ 舒芙蕾白整体基底。  
- **主要文字**：深炭灰 / 焦糖褐，用于标题与商品名，确保在浅色背景上的可读性。  
- **强调色**：
  - 暖金 / 琥珀黄：用于“加入购物车”按钮、价格高亮、选中图标；  
  - 柔和粉 / 橙色：作为“晚霞推荐”等标签色。  
- **图标**：默认深灰；太阳图标及当前选中项使用暖金色高亮。

##### （4）界面布局与功能模块

1. **顶部状态与导航栏（Top Bar）**  
   - 左侧：品牌标识“朝暮 Dawn & Dusk”。  
   - 右侧：  
     - “光阴值 320”+ 沙漏图标；  
     - 用户头像（圆形或圆角）；  
     - 模式切换开关：胶囊状按钮，太阳图标高亮，表示处于日间模式。

2. **头部推荐区（Hero Banner）**  
   大号横向毛玻璃卡片，占据视觉中心。  
   - 左侧：  
     - 标题：“朝·醒神推荐（Morning Awake）”；  
     - 副标语 / Slogan：“把时间，浪费在美好的朝暮里”。  
   - 右侧：  
     - 3 枚竖向推荐商品卡片（如热卡布奇诺、冰凉拿铁、根橙果汁）；  
     - 每张卡片包含商品图、名称与暖金色“加入购物车”按钮。

3. **分类导航栏（Category Tabs）**  
   - 采用胶囊状 pill 按钮横向排列；  
   - 示例分类：**人气 / 醒神早餐 / 咖啡 / 甜点**；  
   - 当前选中项（如“醒神早餐”）背景略高亮、文字使用主色，其余为中性灰。

4. **商品列表区（Product Grid）**  
   - 布局：双列卡片式列表，随屏幕宽度自适应。  
   - 单个卡片内容：  
     - 左侧：商品大图；  
     - 右侧：商品名称（如“经典牛角三明治”）、英文名、价格（如“¥32.00”）、特殊标签（如粉色“晚霞”）。  
   - 视觉：卡片有柔和投影和发光边框，使其从背景中脱颖而出；悬停与点击态可稍微放大 + 阴影加深。

5. **底部导航栏（Bottom Navigation）**  
   - 悬浮于页面底部的全宽毛玻璃横条；  
   - 导航项：**首页 / 点餐 / 订单 / 我的**；  
   - 当前页“首页”图标上方有短横线指示，图标与文字高亮。

#### 2.4.3 夜间模式首页 UI 设计（Dusk Mode）

- **模式名称**：夜间模式（Dusk Mode）  
- **主题**：暮·微醺（Evening Tipsy）

背景为夜晚咖啡厅十足氛围感的实景：窗外深蓝夜幕与城市灯火，室内柔和暖色灯光。

##### （2）视觉风格与氛围

- 氛围关键词：**静谧、深邃、柔和光感、沉浸式**。

##### （3）配色方案（Dusk 首页）

在 2.2.2 基础上，补充夜间首页关键元素色彩：

- **容器背景**：深蓝 / 深灰毛玻璃卡片 + 黛蓝整体背景。  
- **主要文字**：浅灰白（Off-white），保证暗背景上的舒适阅读；  
- **强调色**：  
  - 淡金 / 月光银：用于按钮、积分数字、选中图标，模拟灯光和月光；

##### （4）界面布局与功能模块

整体布局与日间模式完全一致，主要为内容与配色切换：

1. **顶部状态与导航栏**  
   - 品牌文案与结构与日间一致；  
   - 文本与图标切换为浅色；  
   - 模式开关显示月亮图标高亮，表示处于夜间模式。

2. **头部推荐区（Hero Banner）**  
   - 标题变更为：“暮·微醺推荐（Evening Tipsy）”；  
   - 右侧推荐饮品更偏向夜间场景：如气泡水、无酒精特调、晚安牛奶等；

   - 胶囊按钮改为深色玻璃材质；

## 3 业务功能模块命名与体验包装（Business Modules）

本系统在功能上是标准的“在线点餐 + 订单管理 + 会员积分”业务，但在命名与文案上引入“时间感”与“光阴”的概念，使用户在使用过程中感受到更强的品牌氛围和情绪价值。

### 3.1 动态菜单体系（Time-Based Menu）

前端 Vue 根据时间段高亮显示不同分类，后端 Spring Boot 根据时间段推荐不同商品。

推荐时间段设计示例：

- **06:00 - 11:00【朝·醒】(Morning Awake)**  
  推荐：美式、浓缩、醒神早餐组合等提神类饮品 / 餐点。  

- **11:00 - 17:00【午·憩】(Noon Break)**  
  推荐：手冲、拿铁、精致甜点等适合午后休憩的组合。  

- **17:00 - 24:00【暮·微醺】(Evening Tipsy)**  
  推荐：气泡水、无酒精特调、晚安牛奶等偏放松、助眠或适合晚上饮用的饮品。

前端首页“为你推荐”区域调用专门的推荐接口（例如 `/api/products/recommend`），由后端根据当前时间和配置返回对应时段的商品列表，并在文案上切换为“朝·醒推荐”“午·憩限时菜单”“暮·微醺夜饮”等。

### 3.2 会员与积分体系（Loyalty System）

#### 3.2.1 积分命名

- 系统积分对用户展示时统一命名为 **“光阴值”**，用来体现“你在朝暮积累的美好时间”。  
- 示例文案：  
  - “您已在朝暮积累光阴 320 分。”  
  - “本次消费获得光阴值 +28。”

#### 3.2.2 会员等级命名

以一天中时间的推移，隐喻用户与品牌关系的成长：

- Lv1 **晨曦（Dawn）**：初次相遇，如晨光微露。  
- Lv2 **烈阳（Sunlight）**：逐渐熟悉，热情如日当空。  
- Lv3 **晚霞（Sunset）**：长久陪伴，如晚霞般绚烂。  
- Lv4 **繁星（Starry）**：成为知音，如星光般恒久。

**实现建议：**

- 数据库 `t_user` 表中使用数值字段 `level` 存储等级（1/2/3/4）。  
- 前端根据 `level` 数值进行文案和图标映射，例如：  
  - 1 → 晨曦 + 柔和太阳图标；  
  - 4 → 繁星 + 星星图标。  
- 在积分中心、个人主页等区域展示用户等级与下一等级所需光阴值距离，增强成长激励感。

## 4 技术实现与数据库设计（Tech & DB）

### 4.1 动态主题实现（前端）

- 使用 Vue 3 + Vite 作为前端技术栈，配合 Pinia 进行主题状态管理。  
- 在全局样式中定义 CSS 变量，例如：  
  - `--primary-color`  
  - `--bg-color`  
  - `--card-bg-color`  
  - `--text-color-primary`  
  - `--text-color-secondary`  
- 通过 JavaScript 根据当前主题（`dawn` / `dusk`）切换变量值，并添加 `transition` 使切换过程平滑。  
- 主题模式字段示例：  
  - `themeMode: 'auto' | 'dawn' | 'dusk'`  
- 当 `themeMode` 为 `auto` 时，根据当前时间判断处于哪个时间段（如朝 / 暮），自动应用对应主题。界面右上角提供手动切换按钮（☀️ / 🌙），用户操作时将 `themeMode` 改为指定值并持久化到 `localStorage`。

### 4.2 时间感知推荐算法（后端）

- 后端采用 Spring Boot + MyBatis-Plus，实现商品和时间段配置的查询与过滤。  
- 新增 `t_product_time_slot` 表，用于存储商品与时间段（如 `dawn` / `midday` / `dusk`）的关联关系，包含起止时间和推荐优先级字段。  
- 推荐接口示例：  
  - `GET /api/products/recommend`  
- 后端根据服务器当前时间，计算出所处的时间段 code（如 `dawn`），从 `t_product_time_slot` 过滤出匹配商品并按 `priority` 排序。  
- 通过在接口响应中返回当前时间段信息（如“朝·醒”），前端可在页面文案中动态展示不同的区块标题和说明。

### 4.3 积分与会员等级的实现对接

- 积分（光阴值）相关表：  
  - `t_points_account`：记录每个用户的积分账户信息；  
  - `t_points_history`：记录每一次积分变动明细（获得、消耗、退款扣减等）。  
- 在订单完成时，根据订单实付金额与 `t_points_rule` 中的规则计算应获得的积分值，并写入 `t_points_history`，累加更新账户表。  
- 会员等级升级逻辑：  
  - 可基于用户累计消费金额或累计获得积分来划分等级门槛；  
  - 通过定时任务或订单完成时即时计算，自动更新 `t_user.level` 字段。  
- 前端根据 `level` 渲染不同等级名称及对应图标文案。

## 5 毕设意义与创新点阐述（Thesis Justification）

以下内容可直接作为论文中“研究意义”“系统特色”部分的描述基础：

“朝暮”不仅仅是一个商业品牌，更代表了本系统对于 **“数字化陪伴”** 理念的探索。

传统的餐饮点餐系统界面往往较为静态、功能偏重业务本身，缺乏情绪与时间维度的设计。本系统在此基础上进行了以下创新：

- **在前端界面层面**：  
  - 创新性地引入基于时间维度的动态 UI 渲染机制；  
  - 通过 Vue 动态主题与 CSS 变量技术，根据用户所在时段自动在“朝模式（暖阳金）”与“暮模式（黛蓝）”之间平滑切换；  
  - 从视觉心理学角度契合用户不同时间段的生理与情绪节律，提升使用舒适度与沉浸感。

- **在业务与推荐逻辑层面**：  
  - 引入按时间段的商品推荐策略：清晨侧重提神饮品与早餐，白天侧重工作 / 学习场景的咖啡与甜点，夜间则推荐助眠特调与轻饮；  
  - 通过 `t_product_time_slot` 等结构化配置，保证推荐规则具备可扩展性与可维护性。

- **在用户成长与积分体系方面**：  
  - 将抽象的“积分”具象化为“光阴值”，通过“晨曦、烈阳、晚霞、繁星”等等级命名，将会员成长过程与一天中不同时间段进行隐喻结合；  
  - 使用户在消费行为中感受到“与品牌共同度过时间”的情感链接。

综上，本系统尝试在“技术 + 体验 + 品牌”的交叉点上进行探索，以期在满足基本业务需求的同时，为用户提供更具温度与记忆点的在线点餐体验。
