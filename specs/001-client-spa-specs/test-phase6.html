🌅 Phase 6 手动测试清单

请按照以下步骤测试 M1 阶段 6 的所有功能。

准备工作

  • 启动开发服务器: npm run dev
  • 打开浏览器访问: http://localhost:5174
  • 打开浏览器开发者工具 (F12)

T027: 404 页面测试

测试步骤

  • 访问不存在的路由: http://localhost:5174/this-does-not-exist
  • 验证显示 404 页面
  • 验证文案："生活总有遗憾，就像这杯咖啡还没做好。"
  • 点击"回到首页"按钮，验证跳转到首页

✅ 预期结果: 显示品牌化的 404 页面，可以返回首页

T028: Loading 组件测试

测试步骤

  • 访问登录页面: http://localhost:5174/login
  • 输入错误的用户名密码并提交
  • 观察是否显示 Loading 状态
  • 在开发者工具中检查 Network 面板，确认请求发送

✅ 预期结果: 提交表单时显示 Loading 动画

T029: 路由守卫测试

测试 1: 未登录访问受保护路由

  • 确保未登录状态 (清除 localStorage 或使用无痕模式)
  • 访问: http://localhost:5174/profile
  • 验证自动重定向到: /login?redirect=/profile

测试 2: 登录后自动跳转

  • 在登录页输入正确的用户名密码
  • 提交登录
  • 验证自动跳转回 /profile

测试 3: 已登录访问登录页

  • 确保已登录状态
  • 访问: http://localhost:5174/login
  • 验证自动重定向到首页 /

✅ 预期结果: 路由守卫正确保护会员专属路由

T030: 主题持久化测试

测试 1: 主题切换与持久化

  • 在页面头部找到主题切换按钮
  • 切换到"暮"主题，观察页面变暗
  • 刷新页面 (F5)
  • 验证主题保持为"暮"

测试 2: localStorage 验证

  • 打开浏览器控制台 (F12)
  • 执行: localStorage.getItem('dawn_dusk_theme_mode')
  • 验证返回值为 'dawn'、'dusk' 或 'auto'

测试 3: DOM 属性验证

  • 在控制台执行: document.documentElement.dataset.theme
  • 验证返回值为 'dawn' 或 'dusk'
  • 在控制台执行: document.documentElement.classList.contains('theme-dawn') || document.documentElement.classList.contains('theme-dusk')
  • 验证返回 true

✅ 预期结果: 主题设置在刷新后保持，localStorage 和 DOM 状态一致

综合测试

完整用户流程

  • 访问首页，切换主题到"暮"
  • 访问菜单页
  • 尝试访问个人中心 (未登录)
  • 被重定向到登录页
  • 登录成功后自动跳转到个人中心
  • 刷新页面，验证仍然登录且主题保持
  • 访问不存在的路由，显示 404 页面
  • 从 404 页面返回首页

✅ 预期结果: 所有功能协同工作，用户体验流畅
