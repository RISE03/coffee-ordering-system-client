# 首页改造方案（PC 端）——突出“时间”主题（毕设）

> 目标：让首页从“功能页（像点餐页）”升级为“作品展示页（精美好看 + 时间主题强）”。  
> 约束：仅考虑 PC 端视觉与交互；不改后端接口；功能优先级低于观感与叙事。

---

## 为什么要改

当前 `HomeView` 与 `MenuView` 都以“分类 + 商品网格 + 加购”为核心，导致：

- 同质化：首页像“第二个点餐页”，难体现产品差异
- 毕设主题不突出：项目卖点是“时间”，但首页缺少一眼可见的“时间表达”
- 答辩不抓人：难用首页快速讲清“为什么它与时间有关”

---

## 三个可选方案（给你做选择）

下面 3 个方案都以 “时间” 为主轴，区别在于**时间表达方式**与**页面结构**。

### 方案 A：时间主视觉（时间仪表盘 / 时钟装置）【更适合答辩】

**一句话**：首页是一张“时间装置海报”，用大时钟主视觉 + 时段氛围切换，强调“朝暮”概念。  
**关键词**：时间仪表盘、日进度环、朝暮切换、沉浸动效、演示可控

**布局结构示意（PC）**

```text
┌────────────────────────────────────────────────────────────────────────┐
│ HomeHeader：Logo｜光阴值｜主题切换｜头像                                  │
├────────────────────────────────────────────────────────────────────────┤
│ 左侧：时段主标题/情绪文案                 右侧：TimeDial（日进度环/时钟装置）│
│  "此刻·午后" / "把时间浪费在美好里"       中心：HH:mm / 日期 / 星期         │
│  [ 去点餐 / Explore ]                    外环：旋转光晕/粒子/高光追踪       │
│                                                                        │
│ 下方：此刻推荐（0~3 张玻璃卡点缀）        侧栏：演示控制台（开关/拖动时间轴）│
├────────────────────────────────────────────────────────────────────────┤
│ BottomNav（可保留但弱化，PC 以主视觉为主）                                 │
└────────────────────────────────────────────────────────────────────────┘
```

**PC 页面结构（建议）**
- 主视觉（Hero）：大时钟/日进度环（占屏 60% 视觉权重）
- 时间信息：当前时间/日期/星期（大字号排版）
- 时段叙事：朝/昼/暮（对应文案与微动画）
- “此刻推荐”：展示 0~3 个推荐卡（用于点缀，不做全量商品网格）
- 演示控制台（答辩用）：拖动时间轴、是否同步主题开关、快速跳转到“朝/昼/暮”按钮

**优点**
- 一屏就能讲清“与时间有关”，最适合毕设展示
- 视觉冲击力强，容易做出“精美好看”的作品感

**注意点**
- 动效要“精而不乱”：关键动效集中在主视觉与主题切换

---

### 方案 B：时间线叙事（从清晨到夜晚的故事页）

**一句话**：首页是一条“时间线”，从清晨到深夜讲完整天的消费场景与情绪。  
**关键词**：时间线分段、故事文案、分区大图、轻功能

**布局结构示意（PC）**

```text
┌────────────────────────────────────────────────────────────────────────┐
│ HomeHeader：Logo｜主题切换｜头像                                          │
├────────────────────────────────────────────────────────────────────────┤
│ 首屏：全屏沉浸背景（窗外光影/雨滴/阳光）                                  │
│  超大背景字：Morning Awake / 此刻·10:00                                  │
│  极简玻璃信息条：时间｜时段｜一句话情绪文案                               │
│  精选点缀：3 张玻璃卡（横排/错位） + 1 个主 CTA（去点餐）                 │
│  ↓ Scroll：提示“时间在流逝”                                              │
├────────────────────────────────────────────────────────────────────────┤
│ 向下滚动：时间线分区（朝 → 午 → 暮 → 夜）                                 │
│  [大图+文案+CTA]  [大图+文案+CTA]  [大图+文案+CTA]  [大图+文案+CTA]      │
└────────────────────────────────────────────────────────────────────────┘
```

**PC 页面结构（建议）**
- 纵向滚动时间线：朝/午/暮/夜 4 段
- 每段：1 张大图 + 1 句文案 + 1 个主 CTA（去点餐/看推荐）
- 每段点缀：1~2 个精选卡（不做全量商品浏览）

**优点**
- 叙事强，适合“作品阐述”
- 动效可以更克制，开发成本可控

**风险**
- 对图片/文案质量依赖较高，否则容易显得“像信息流”

---

### 方案 C：时间胶片画廊（Dawn/Dusk 画廊与光影）

**一句话**：首页像“时间胶片墙”，用图像与材质表达时间流动与对比。  
**关键词**：画廊墙、胶片质感、光影、视差、氛围

**布局结构示意（PC）**

```text
┌────────────────────────────────────────────────────────────────────────┐
│ HomeHeader：Logo｜主题切换｜头像                                          │
├────────────────────────────────────────────────────────────────────────┤
│ 左侧：时间胶片条/刻度（Dawn｜Dusk｜All）   右侧：画廊墙（栅格/瀑布流）      │
│  - 当前时刻标记（●）                       [卡片][卡片][卡片][卡片]       │
│  - 轻文案：此刻·xx:xx                       [卡片][卡片][卡片][卡片]       │
│  - CTA：去点餐                              点击卡片 → 弹层：时段含义+推荐  │
├────────────────────────────────────────────────────────────────────────┤
│ BottomNav（可保留但弱化）                                                  │
└────────────────────────────────────────────────────────────────────────┘
```

**PC 页面结构（建议）**
- 大背景 + 画廊栅格/瀑布流（朝/暮两套视觉混排或分区）
- 卡片点开：弹层展示“此刻含义 + 推荐”

**优点**
- 氛围感最强，观感容易高级

**风险**
- 素材不足时会失去优势
- 交互做不好会变成“只有好看，没有信息”

## 方案 A 落地（可开发级实现蓝图）

### 目标与验收标准

- 首页不再展示“分类 Tabs + 全量商品网格”（避免与 `/menu` 重复）
- 首屏必须一眼可见：
  - **时间主视觉装置**（时钟/日进度）
  - **时段（朝/昼/暮）**与对应文案
  - **主题氛围切换**（Dawn/Dusk）
- 演示模式（答辩用）：
  - 可拖动时间并实时更新主视觉与时段文案
  - 可选择是否同步全局主题（只用于预览，不落库）
- 推荐区仅展示 0~3 个推荐商品（点缀即可）

### 建议新增/修改的文件（路径）

**新增（组件）**
- `src/components/home/TimeHero.vue`：首页时间主视觉容器（整屏排版 + 文案 + 推荐）
- `src/components/home/TimeDial.vue`：时间仪表盘（SVG/Canvas 均可，优先 SVG）
- `src/components/home/TimeDemoPanel.vue`：演示控制台（时间滑杆/快速切时段/同步主题开关）

**修改（页面/状态/样式）**
- `src/views/HomeView.vue`：改为 TimeHero 为核心，移除商品网格与分类筛选逻辑
- `src/stores/theme.ts`：增加“预览主题”能力，仅供演示模式使用（不写 localStorage）
- `src/style.css`：增加主视觉光环/渐变环/浮动光晕等动效（需注意性能）

> 注：以上是“可实现蓝图”，具体开发时可按现有组件风格拆分/复用。

### 首页时间来源（真实 vs 演示）

- 默认：使用系统真实时间（每秒刷新显示）
- 演示模式：使用用户选择的“演示时间”（小时/分钟），用于现场展示

### 时段划分（用于标题与推荐）

建议在首页内固定划分（与推荐参数对齐）：

- `06:00-11:59` → `dawn`（朝）
- `12:00-17:59` → `midday`（昼）
- `18:00-05:59` → `dusk`（暮）

当跨段（例如 11:59→12:00）时刷新一次推荐即可，避免频繁请求。

## PC 验收清单（简版）

- 首页首屏：强“时间感”，视觉精美，动效流畅（无明显卡顿）
- 首页不再是“点餐页复制品”（无全量商品网格）
- 演示模式可用：拖动时间→主视觉与时段立即变化；可开关同步主题
